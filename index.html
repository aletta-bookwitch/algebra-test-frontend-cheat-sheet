1. HTML

Even though our code will least of all consist of HTML, we can't have a web page without it. ^^
We need to tell browsers what kind of document they will be loading, and give it some kind of structure...

What is HTML?
  • HTML stands for Hyper Text Markup Language.
  • HTML is the standard markup language for creating Web pages.
  • HTML describes the structure of a Web page.
  • HTML elements tell the browser how to display the content.

Basic HTML document structure

To create a HTML we need to define the following:
  1. The type of document (HTML).
  2. The "head" of the document.
  3. The "body" of the document.
HTML consists of elements such as:
  • Heading (<h1> </h1>, <h2> </h2>, etc.)
  • Paragraph (<p> </p>)
  • Empty element (<br>)

This part is the "skeleton" of our code, and it looks like this: 


            <!DOCTYPE html>
             <html>
              <head>
               (We name it here, and it doesn't show on our page. You can put a title in here, and that will be the tab name. Yay!)
              </head>
                <body>
                 Some kind of text. (This is what will be shown on the page. Definitely. Unless you hide it somehow. But why would you do that?)
                </body>
             </html>
We can form segments of text either by typing it as a paragraph, or make a block of it by turning it into a <div>.

Like this: 
        <p>
            Lots of text here. Probably useless, but still fun. At least to someone. You never know who might be reading your page. 
            It could be your cat. You think they're harmless, but believe me, they will rule the world at some point. 
            You can see it in their eyes. The determination. Or it could just be hunger. Who knows.
        </p>
Or... 
            
        <div>
            Lots of text here. Probably useless, but still fun. At least to someone. You never know who might be reading your page. 
            It could be your cat. You think they're harmless, but believe me, they will rule the world at some point. 
            You can see it in their eyes. The determination. Or it could just be hunger. Who knows.
        </div>

Lists

We can make unordered, ordered, and description lists.

This is an unordered list.
  • This is one segment in a list.
  • This is another segment in a list.
  • This is another segment in a list.

<ul>
  <li>This is one segment in a list.</li>
  <li>This is another segment in a list.</li>
  <li>This is another segment in a list.</li>
</ul>

This is an ordered list.
  1. This is the first segment in a list.
  2. This is the second segment in a list.
  3. This is the third segment in a list.

<ol>
  <li>This is the first segment in a list.</li>
  <li>This is the second segment in a list.</li>
  <li>This is the third segment in a list.</li>
</ol>

This is a description list.
strong
Used for important text.
em
Used for good-to-know text.
small
Used for footer text.

Images


We can insert images which are also links to other pages.


<a href="https://www.youtube.com/watch?v=uHKfrz65KSU" target="_blank"> <img src="./images/cute-kitty.gif" alt="Tiny cute-kitty" height="100"> </a>


We can also "draw" using svg (scalable vector graphics):

<svg width="100" height="100">
  <ellipse cx="50" cy="50" rx="30" ry="15" stroke="red" fill="green" stroke-width="3"/>
</svg>

<svg width="200" height="200">
  <rect x="50" y="50" width="130" height="130" stroke="black" fill="black" stroke-width="1"/>
  <circle cx="115" cy="115" r="32.5" stroke="red" fill="white" stroke-width="3"/>
</svg>
